# Hosted Tableau MCP Transport Skeleton

## 1. Purpose

This document defines the Hosted Tableau MCP transport skeleton added for v0.8.0.

The skeleton exists so later hosted execution work can depend on a safe transport boundary without requiring live network access, tokens, or production Hosted MCP behavior.

## 2. Background

v0.7.0 established the transport abstraction, configuration strategy, auth boundary, fake handlers, and transport-aware metadata execution boundary.

v0.8.0 moves toward Hosted / remote execution, but this issue only adds the transport shell needed for later hosted wiring.

## 3. Transport contract alignment

The skeleton aligns with the existing `TableauMcpTransport` contract:

- `kind` is `hosted`
- `call(request)` returns a `TableauMcpTransportResult`
- supported status values are represented safely
- timing and trace metadata are always JSON-safe

## 4. Skeleton responsibilities

The skeleton is responsible for:

- accepting config and dependencies through the constructor
- returning `not_configured` when endpoint, auth, or other required inputs are missing
- returning `unsupported` for unsupported tools or operations
- returning safe timeout / failed results for injected client failures
- avoiding real network access by default

## 5. Configuration boundary

The skeleton does not read `process.env` directly.

Configuration is provided through:

- endpoint
- auth mode
- token reference
- site context
- timeout
- network enabled flag
- protocol hint

## 6. Dependency injection boundary

The skeleton accepts injected dependencies:

- request client
- clock
- logger

This keeps the transport testable without live Hosted MCP access.

## 7. not_configured behavior

Return `not_configured` when:

- endpoint is missing
- auth is missing
- site context is required but missing
- network access is disabled
- request client is missing when a live call would be needed

The result must stay safe and must not include raw tokens or secrets.

## 8. unsupported behavior

Return `unsupported` when the requested tool or operation is not allowed by the skeleton.

This is a placeholder for later tool-mapping work and should not expose raw Hosted MCP details.

## 9. Timeout behavior

Timeout handling is placeholder-only in this issue.

The skeleton should:

- respect request timeout values
- treat invalid timeout values as safe failure / not configured
- map timeout-like client errors to `TIMEOUT`
- avoid real abort-controller network handling

## 10. Error mapping placeholders

The skeleton should safely map or normalize:

- `TRANSPORT_NOT_CONFIGURED`
- `AUTH_REQUIRED`
- `AUTH_EXPIRED`
- `PERMISSION_DENIED`
- `SITE_SETTINGS_DISABLED`
- `NETWORK_ERROR`
- `TIMEOUT`
- `MCP_PROTOCOL_ERROR`
- `TOOL_NOT_FOUND`
- `INVALID_TOOL_INPUT`
- `REMOTE_SERVER_ERROR`
- `UNSUPPORTED_TRANSPORT`
- `UNKNOWN_ERROR`

## 11. No-network default behavior

The default constructor must not make a live network call.

No Hosted MCP endpoint, OAuth flow, token storage, or Tableau Cloud access is required for the skeleton tests.

## 12. Token and secret handling

The skeleton must not log or return:

- access tokens
- refresh tokens
- authorization headers
- secret values

If logging is used, it must stay summary-only.

## 13. Trace and timing metadata

The skeleton should attach safe trace and timing fields such as:

- requestId
- correlationId
- agentRunId
- toolName
- transportKind
- startedAt
- completedAt
- durationMs
- timeoutMs
- endpointConfigured
- authConfigured
- networkEnabled
- requestClientConfigured

## 14. Relationship with integration test gating

The skeleton is designed to work with the Hosted integration test gating policy in `docs/v0.8-hosted-mcp-integration-test-gating.md`.

It must remain no-network friendly and should not force default CI to require Hosted MCP.

## 15. Relationship with auth context adapter

The skeleton expects auth context to be passed in safely by later adapter work.

It must preserve the tokenReference boundary and avoid direct token handling.

## 16. Relationship with describeDatasource hosted execution

This issue does not connect `tableau.metadata.describeDatasource` to the transport yet.

That wiring belongs to #161.

## 17. Non-goals

This issue does not include:

- production Hosted MCP execution
- real network calls
- OAuth flow
- token storage or refresh
- Hosted integration test execution
- metadata tool wiring

## 18. Open questions

Open questions remain for:

- the final Hosted MCP endpoint shape
- exact auth flow requirements
- site context requirements
- supported protocol details
- later tool mapping
