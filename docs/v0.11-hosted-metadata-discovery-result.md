# v0.11.0 Hosted Metadata Discovery Result

## Summary

v0.11.0 made Hosted Tableau MCP usable in the existing structured metadata discovery flow, limited to safe read-only `describeDatasource` execution.

The milestone did not broaden Hosted MCP into a generic remote execution surface. It kept the hosted path opt-in / gated, preserved fake and stdio fallback behavior, and kept default validation no-network.

## Milestone outcome

The v0.11.0 issue set reached the following outcome:

- #236 confirmed and finalized the `docs/v0.11-plan.md` boundary
- #237 implemented the hosted `describeDatasource` path behind the existing metadata discovery boundary
- #238 records the result and wrap-up

The concrete result is:

- eligible `metadata_discovery` datasource plans can reach Hosted execution when the hosted feature flag and configuration are enabled
- Hosted execution is still limited to `describeDatasource`
- unsupported, ambiguous, or unsafe requests stay out of Hosted execution
- fake / stdio fallback remains available
- default no-network behavior remains intact

## What is now possible

v0.11.0 allows the application to:

- keep the existing `metadata_discovery` flow
- send eligible datasource metadata requests through a gated Hosted path
- use the Connected App + Direct Trust boundary for hosted metadata requests
- normalize successful Hosted results before user-facing output
- preserve safe fallback behavior for disabled, not configured, and unsafe cases

In practice, the milestone turns Hosted Tableau MCP from a connection-only proof into a usable metadata discovery path for the safe `describeDatasource` case.

## What is enabled and gated

Enabled when the hosted path is explicitly configured:

- hosted metadata discovery for eligible datasource plans
- Direct Trust auth context preparation
- hosted `describeDatasource` execution
- safe normalization of success and failure cases

Still gated or disabled by design:

- hosted execution when the feature flag is off
- hosted execution when config is missing
- hosted execution when the request is not an eligible metadata discovery plan
- hosted integration tests unless explicitly enabled
- any broader hosted expansion beyond `describeDatasource`

## Fallback behavior

Fallback behavior remains intentionally conservative:

- fake / no-network execution stays first-class
- stdio fallback stays available
- `not_configured` is returned safely when hosted configuration is incomplete
- auth failure, permission denial, timeout, and transport failures are normalized instead of being exposed raw
- broad free-form chat routing is unchanged

The milestone does not require real hosted credentials for the default validation path.

## Validation summary

The code-bearing implementation in #237 was validated with the repository's local quality gates:

- `npm run quality:precommit`
- `npm run quality:prepush`
- `npm run test:e2e`
- Gitleaks secret scan

This docs-only wrap-up follows the repository docs policy and does not add new runtime validation requirements.

## Remaining limitations

v0.11.0 intentionally stops short of:

- `listFields` execution
- arbitrary query generation or execution
- underlying data access
- field values or row data retrieval
- write-capable tools
- DCR-dependent OAuth 2.1 Remote MCP host flow
- token storage or refresh
- broad ChatService replacement
- broad free-form chat routing rewrite
- Exploration Session
- AgentCore

The hosted path is limited to safe metadata discovery, not general-purpose remote Tableau access.

## Remaining risks

The main remaining risks are:

- hosted endpoint behavior may vary from local assumptions
- token presentation for Hosted MCP must continue to be verified when the integration evolves
- permission-denied and timeout handling must remain conservative
- future expansion beyond `describeDatasource` may widen the test matrix
- hosted integration tests remain gated and need explicit enabling

## Next possible expansion

The next milestone, if any, should be planned separately and should not assume broader hosted access by default.

Likely candidates include:

- `listFields` readiness review
- broader hosted metadata expansion only after separate planning
- further validation of hosted auth presentation details
