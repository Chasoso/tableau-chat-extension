# v0.10.0 LLM ResponseComposer Result

## Summary

Branch A for v0.10.0 delivered the LLM ResponseComposer result while keeping the deterministic composer as the baseline and fallback.

LLM composition remains optional and guarded. The response material contract, fake no-network composer behavior, fallback rules, and regression coverage together keep the feature narrow and auditable.

Real LLM / hosted dependency support is not required for default tests. Raw MCP / raw transport / raw Tableau response material is not exposed to the composer.

Exploration Session remains deferred, and AgentCore remains deferred.

## Relationship to v0.10.0 branch decision

#209 selected Branch A: LLM ResponseComposer and deferred Branch B: Exploration Session.

That decision was made to improve response quality first while keeping the structured discovery boundary from v0.9.0 intact.

The branch decision intentionally avoided widening session state, lifecycle, privacy, or stale-context scope.

## What was enabled

The Branch A work enabled:

- the guarded LLM ResponseComposer prototype
- feature-flag and config-gated composition behavior
- a fake no-network composer path for testing
- integration with the safe response material contract
- output validation before composition results are accepted
- deterministic fallback behavior
- safe fallback reason codes
- regression coverage for no-network, timeout, error, invalid-output, and unsafe-output paths

## What remained disabled or gated

The following remained disabled, gated, or deferred:

- LLM composition is optional and guarded
- deterministic composer behavior remains the default baseline
- real LLM / hosted dependency is not required for default paths
- real LLM / hosted dependency is not required for default tests
- production-grade provider integration remains gated or deferred
- Exploration Session remains unimplemented
- AgentCore remains unimplemented
- broad legacy chat replacement did not happen
- `listFields` execution did not happen
- arbitrary query generation and execution did not happen

## Deterministic baseline result

The deterministic ResponseComposer remains the baseline for all guarded composition paths.

If LLM composition is disabled, missing, timed out, invalid, unsafe, or otherwise unavailable, the system falls back to the deterministic composer.

Required limitation notes are preserved, and raw error or stack trace details are not surfaced to users.

## LLM composition boundary result

#210 defined the safety boundary for LLM composition.

That boundary kept composition:

- optional and guarded
- bounded to safe input material
- limited to user-facing wording and explanation work
- separate from tool execution decisions
- free of LLM-generated Tableau MCP queries
- free of arbitrary query generation
- free of raw MCP / raw transport exposure

## Response material contract result

#211 defined the safe response material contract used by the composer.

The contract established:

- allowed safe material sources
- prohibited raw material sources
- JSON-safe response material shape
- evidence and citation discipline
- facts versus limitations separation
- required disclosures
- redaction and suppression rules
- safe handling for `selected_mark_explanation`
- safe handling for `metadata_discovery`

## Guarded prototype result

#212 added the guarded prototype that consumes the safe response material contract.

The prototype uses:

- feature-flag and config gating
- fake composer behavior for no-network tests
- output validation before accepting composed output
- safe fallback reason handling
- deterministic fallback when composition is unavailable or unsafe

The prototype stayed narrow and did not introduce a general LLM integration surface.

## Fallback and regression result

#213 added fallback and regression coverage for the guarded prototype.

That coverage protects:

- disabled and not-configured behavior
- timeout and error behavior
- invalid output behavior
- unsafe output behavior
- too-long and missing-limitation-note behavior
- trace-safe fallback reasons
- no-network default posture

The result confirms that failures in composition do not alter the deterministic fallback path.

## selected_mark_explanation result

`selected_mark_explanation` remains protected.

Its deterministic behavior and output shape are unchanged, and selected-mark raw data, row values, and field values are not forced through the LLM composer.

The optional composer does not replace the existing selected-mark path.

## metadata_discovery result

`metadata_discovery` can use safe response material where applicable, but only through the normalized and bounded contract.

The result keeps raw `describeDatasource` output, raw `listFields` output, raw MCP output, and raw transport output out of the composer input path.

Truncation, permission, and limitation notes remain preserved, and unknown counts or permissions remain unknown instead of being fabricated.

## Legacy fallback result

Legacy free-form chat fallback remains available.

The Branch A work did not replace `ChatService`, did not retire `runLightweightAgentLoop`, and did not force all non-discovery requests through the composer.

Broad free-form chat replacement remains out of scope.

## No-network posture

Default tests remain no-network.

The fake composer path allows validation of guarded composition behavior without requiring real LLM secrets or hosted dependencies.

Hosted or real LLM integration tests remain optional or gated rather than required by default.

## Safety and evidence discipline

The Branch A result preserves the following boundaries:

- no raw MCP tool exposure
- no raw MCP output exposure
- no raw transport output exposure
- no raw Tableau response exposure
- no arbitrary query generation
- no arbitrary query execution
- no underlying data access
- no field values or row data retrieval
- no sample values or domain values retrieval
- no write-capable tools
- no tokens, secrets, or stack traces in user-facing output or traces
- evidence-bounded response material only
- required limitation notes preserved
- unknown remains unknown
- deterministic fallback on failure, timeout, or unsafe output
- `selected_mark_explanation` remains protected
- legacy free-form chat fallback remains available
- Exploration Session remains deferred
- AgentCore remains deferred

## Remaining risks

The remaining risks are mainly future-facing:

- structural output validation may not catch every unsupported claim
- a real provider may behave differently from the no-network fake path
- prompt and material shaping may need tuning after real usage
- evidence discipline will need to stay enforced as composition broadens
- future application to other response types may need additional care
- timeout, cost, IAM, and observability decisions still matter for real integration

## Deferred work

The following remains deferred beyond this Branch A result:

- production-grade real LLM provider integration
- broader LLM composition coverage
- `selected_mark_explanation` LLM composition if it is ever enabled
- legacy chat LLM composition if it is ever enabled
- Exploration Session
- AgentCore
- `listFields` execution
- arbitrary query generation and execution
- underlying data access
- field values or row data access
- write-capable tools
- frontend redesign
- default hosted or real LLM CI
