# Metadata Output Normalization and Trace Events

## 1. Purpose

This document defines the safe output boundary for `tableau.metadata.describeDatasource` and `tableau.metadata.listFields`.
It explains how raw transport results are normalized before they are exposed to `ToolExecutionWrapper`, `ResponseComposer`, or trace payloads.

## 2. Background

v0.7.0 introduces metadata tools as read-only wrapper tools, not raw MCP exports.
By the time output leaves the execution boundary, it must already be summary-first, JSON-safe, and free of secrets or raw stack traces.

## 3. Normalization principles

- Normalize after transport execution and before user-facing presentation.
- Keep the distinction between raw transport result and safe normalized result.
- Prefer allowlisted summary fields over copying raw result blobs.
- Preserve bounded warnings, errors, truncation, omissions, and resolution state.
- Treat precondition-blocked execution as a distinct normalized state.

## 4. Normalized metadata result shape

The normalized result is a safe wrapper around the metadata tool output.

- `toolName`
- `status: success | partial | failed | blocked`
- `summary`
- `warnings`
- `error`
- `truncation`
- `omissions`
- `resolution`
- `trace`
- `metadata`

The `summary` field is tool-specific:

- `describeDatasource` uses datasource summary plus supporting metadata such as fields and connection summaries.
- `listFields` uses datasource summary plus a bounded field list and field-count summary.

## 5. Raw result suppression

Raw transport-level fields are never surfaced directly.
The normalization layer suppresses or ignores fields such as:

- `raw`
- `rawResult`
- `rawMcpResult`
- `mcpResponse`
- `serverResponse`
- `transportRawResult`
- `stdout`
- `stderr`
- `stack`
- `stackTrace`
- `accessToken`
- `refreshToken`
- `authorizationHeader`
- `secret`
- `token`

## 6. JSON-safe output boundary

Normalized results and trace events must remain JSON-safe.
Only plain JSON values should survive:

- strings
- numbers
- booleans
- `null`
- arrays
- plain objects
- ISO datetime strings

Unsafe runtime values such as `Date`, `Error`, `BigInt`, `Map`, `Set`, functions, symbols, and circular structures are sanitized or omitted.

## 7. Truncation and omission handling

The normalized result preserves bounded reduction signals:

- `truncation.truncated`
- `truncation.limit`
- `truncation.returned`
- `truncation.totalAvailable`
- `omissions[]`

These signals explain when field lists were shortened or when hidden/technical metadata was omitted.

## 8. Warning and error summaries

Warnings and errors are represented as safe summaries, not raw exceptions.

- Warnings keep a stable code, message, and optional bounded metadata.
- Errors keep a stable code, message, retryability, and user-action hint.
- Timeout, not-configured, auth, and permission failures are normalized into safe error codes.

## 9. Trace event names

The metadata path uses three trace event candidates:

- `tableau_metadata_tool.started`
- `tableau_metadata_tool.completed`
- `tableau_metadata_tool.failed`

## 10. Trace event payloads

Trace events are JSON-safe and bounded.
They may include:

- `requestId`
- `correlationId`
- `agentRunId`
- `toolName`
- `status`
- `transportKind`
- `startedAt`
- `completedAt`
- `durationMs`
- `warningCount`
- `errorCode`
- `truncated`
- `omittedCount`
- `fakeNoNetwork`

Trace events never include raw MCP payloads, tokens, secrets, stdout, stderr, or raw stack traces.

## 11. Trace event builder behavior

The builder layer provides explicit helpers for started, completed, and failed events.
These helpers are intended for tests and for future trace wiring, but they do not persist trace data on their own.

## 12. Execution boundary integration

`#136` produces raw transport results at the execution boundary.
`#137` normalizes those results into a safe metadata response material before it is surfaced.

The execution flow is:

`tool input -> precondition -> transport -> normalized result -> trace summary`

## 13. Separation between transport result and user-facing summary

- Transport results are internal and transport-specific.
- Normalized metadata results are safe, bounded, and user-facing.
- The user-facing response should use normalized summary material, not raw transport payloads.

## 14. Safety boundaries

- No raw transport output exposure.
- No secrets in trace events.
- No raw stack traces in trace events.
- No UI trace panel or trace persistence in v0.7.0.

## 15. Relationship with ResponseComposer

`ResponseComposer` should consume normalized summary material, not transport internals.
The normalized metadata result is designed to be safe enough for future response composition without exposing raw Tableau content.

## 16. Recommendations for #138 and #139

- `#138` should document how normalized metadata results flow into the migration path.
- `#139` should close out the v0.7.0 foundation with a concise summary of what is now safe, testable, and still deferred.

## 17. Open questions

- Should future trace wiring persist the explicit trace event objects, or only their bounded summaries?
- Should the normalized result retain more transport metadata, or keep only the current safe summary fields?
- If real Hosted MCP output differs from the fake path, which fields should remain stable across transports?
