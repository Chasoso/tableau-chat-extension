# Fake No-network and stdio Fallback Regression Coverage

## 1. Purpose

This document preserves the regression boundary for the no-network and fallback
coverage that must remain stable while v0.8.0 Hosted Tableau MCP work lands.

The intent is to keep the following behaviors intact:

- fake / no-network metadata tests
- stdio fallback config tests
- `selected_mark_explanation` regression tests
- ToolRegistry metadata registration tests
- free-form chat legacy non-regression checks
- default CI without Hosted MCP

## 2. Background

v0.8.0 introduces Hosted transport wiring, hosted auth context handling,
feature-flagged `describeDatasource` execution, hosted error normalization, and
hosted trace coverage.

Those changes must not disturb the deterministic baseline that already keeps
local development and default CI safe:

- fake / no-network metadata handling remains the default path
- stdio fallback remains available where configured
- `selected_mark_explanation` continues to use its existing deterministic
  orchestration
- free-form chat remains on the legacy path

## 3. Regression boundary

This issue is about preservation, not expansion.

The regression boundary includes:

- fake metadata execution
- no-network metadata execution
- stdio fallback configuration behavior
- ToolRegistry metadata registration behavior
- selected mark orchestration behavior
- legacy free-form chat behavior
- default CI behavior

Out of scope:

- Hosted MCP live execution
- OAuth flow
- token handling changes
- frontend changes
- new broad tool exposure

## 4. Default no-network test path

The default backend test path should continue to run without Hosted MCP,
OAuth, secrets, Tableau Cloud access, or real network access.

Existing coverage already exercises this path through:

- `backend/test/tableauMetadataToolRuntime.test.ts`
- `backend/test/tableauMetadataTransportBoundary.test.ts`
- `backend/test/tableauMetadataPreconditions.test.ts`
- `backend/test/tableauMetadataOutputNormalization.test.ts`
- `backend/test/tableauMcpContextProvider.class.test.ts`
- `backend/test/toolRegistry.test.ts`
- `backend/test/selectedMarkOrchestration.test.ts`
- `backend/test/responseComposer.test.ts`
- `backend/test/chatAgent.test.ts`
- `backend/test/chatService.finalAnswer.test.ts`

## 5. Fake metadata coverage

The fake metadata path remains the baseline for deterministic execution.

The current coverage verifies that:

- `tableau.metadata.describeDatasource` works without network access
- `tableau.metadata.listFields` works without network access
- fake output is normalized
- fake warnings are preserved
- trace metadata is emitted safely
- raw MCP payload markers do not leak into JSON output

## 6. stdio fallback coverage

stdio fallback remains a supported configuration path.

The current coverage verifies that:

- stdio transport configuration is still recognized
- fallback selection remains governed by resolver and precondition logic
- transport configuration failures remain safe
- fallback behavior does not require live Hosted MCP
- default CI does not need to launch a real stdio process

## 7. ToolRegistry metadata registration coverage

ToolRegistry coverage continues to ensure that metadata tool registration stays
stable.

The current tests verify that:

- `tableau.metadata.describeDatasource` is registered
- `tableau.metadata.listFields` is registered
- raw MCP tool names are not exposed
- tool definitions remain JSON-safe and execution-free
- Hosted feature flags do not remove metadata tool registration

## 8. selected_mark_explanation regression coverage

The `selected_mark_explanation` path must remain deterministic and unchanged.

Existing regression checks cover:

- deterministic orchestration
- placeholder response composition
- tool registry lookups for selected marks
- plan and response composer behavior
- fallback when selected marks are missing

This issue does not change that path.

## 9. Free-form chat legacy non-regression

Free-form chat remains on the legacy path.

Existing coverage verifies that:

- `ChatService` remains available
- `runLightweightAgentLoop` remains available
- legacy chat tests do not require Hosted MCP
- hosted metadata work does not replace the free-form chat path

## 10. Hosted integration exclusion from default CI

Default CI must remain no-network and must not require Hosted MCP secrets or a
live Hosted endpoint.

Hosted integration tests, if present later, remain gated and optional.

## 11. CI and local development expectations

Expected behavior remains:

- default CI uses fake / no-network coverage
- local development can run without Hosted MCP
- stdio fallback stays optional and configuration-driven
- regression coverage must remain deterministic

## 12. Safety and governance boundaries

The regression baseline must keep the existing safety rules intact:

- no raw MCP payload exposure
- no tokens or secrets in test output
- no arbitrary query tools
- no underlying data access
- no write-capable tools
- no frontend trace UI work
