# v0.3.0 Tableau Context First

## Background

v0.2.0 established the agent foundation on the backend side: agent core types, trace schema, `AgentRunner`, `LambdaAgentRunner`, and a minimal fixed plan design.

v0.3.0 shifts the emphasis to Tableau context before LLM orchestration. The goal is to make the extension understand the current dashboard state first, surface that state to the user, and keep the data model aligned with the backend `ContextPack`.

## Goal

Strengthen Tableau context capture and make the pre-LLM context visible in the UI.

The core interaction we want is:

`Tableau mark selection -> extension recognizes current state -> AI Context Preview updates`

## Scope

- Audit current Tableau Extension API usage.
- Add or refine collectors for filters, parameters, selected marks, and summary data preview.
- Define a frontend context preview model that stays aligned with backend `ContextPack`.
- Add an AI Context Preview panel that shows the data before it reaches the LLM.
- Add selected-mark-aware action suggestions such as "explain this selection".
- Keep the context model ready for future orchestration work.

## Out of scope

- Full Intent Resolver implementation.
- Full Plan Builder integration.
- Tool Router implementation.
- Fixed plan execution wiring.
- LLM classification changes.
- Replacing `runLightweightAgentLoop`.
- Large changes to `ChatService`.
- Tool registry work for Tableau MCP.
- AgentCoreRunner.
- AgentCore Gateway.
- Exploration session or memory features.

## Tasks

- Review current Tableau Extension API usage and identify available context signals.
- Define the next-stage context preview experience.
- Collect filters, parameters, selected marks, and summary data preview.
- Keep the frontend preview data structure aligned with backend `ContextPack`.
- Add a read-only AI Context Preview panel.
- Add lightweight action suggestions for selected marks.
- Document the next phase boundary so v0.4 can focus on orchestration.

## Acceptance criteria

- The extension can surface richer Tableau context before AI execution.
- The preview model is compatible with backend `ContextPack` naming and structure.
- The UI shows the current dashboard state in a readable preview panel.
- Selected marks can trigger context-aware suggestions without invoking full orchestration.
- The issue clearly separates v0.3 context work from v0.4 orchestration work.

## Validation

- Review the drafted sub-issues and confirm that each one fits a single PR.
- Confirm the scope does not include orchestration or agent execution rewrites.
- Confirm the issue text is ready to use for GitHub issue creation.
