# Title
v0.2.0 Architecture Reset

## Background
The current Tableau chat PoC works, but the agent flow has grown around Bedrock-centric orchestration. Context collection, intent handling, tool execution, retries, fallback behavior, and trace-like logging are spread across `ChatService`, `ChatAgent`, and `TableauMcpContextProvider`.

We want to reset the architecture around Tableau context first, reduce dependence on free-form agent loops, and create a clear boundary that can later be swapped from `LambdaAgentRunner` to `AgentCoreRunner`.

## Scope
- Define the v0.2 architecture target and migration strategy.
- Establish the new boundaries for Context, Intent, Plan, Tool, and Trace.
- Align backend and frontend responsibilities with the new architecture.
- Identify what should be reused and what should be replaced.

## Out of scope
- AgentCore migration.
- Voice input.
- Full UI redesign.
- Rewriting authentication, deployment, or Tableau extension loading from scratch.

## Tasks
- Review the current implementation and identify the main entry points and dependencies.
- Propose a new architecture centered on Tableau Context.
- Define the directory structure and ownership boundaries.
- Specify the v0.2 minimal scope and the v0.3+ follow-up scope.
- Document the migration approach and open questions.

## Acceptance criteria
- A written architecture reset plan exists in `docs/architecture-reset-v0.2.md`.
- The plan clearly states what will be reused and what will be rebuilt.
- The plan can be used as the source of truth for the next implementation issues.
- The plan explicitly prepares for `AgentRunner` abstraction and traceability by `agentRunId`.
