# v0.6.0 AgentCore Spike

## Background

v0.4.0 delivered structured orchestration for `selected_mark_explanation`.
v0.5.0 delivered the Tool Layer needed to keep that path deterministic, traceable, and compare-ready.

The next step is to define an `AgentRunner` boundary so the current Lambda-based runtime can be compared with a future AgentCore-based runtime without forcing a full migration.

## Goal

Define the v0.6.0 AgentCore Spike scope, boundaries, and follow-up work so the team can compare LambdaAgentRunner and AgentCoreRunner on a narrow selected_mark_explanation path.

## Scope

- AgentRunner abstraction
- LambdaAgentRunner comparison wrapper
- selected_mark_explanation as the first comparison target
- comparison metrics and observability criteria
- AgentCoreRunner design discussion
- follow-up planning for Tableau MCP and LLM response composition

## Out of scope

- AgentCore implementation
- AgentCoreRunner implementation
- replacing ChatService
- replacing runLightweightAgentLoop
- free-form chat migration
- broad Tableau MCP migration
- LLM-based ResponseComposer

## Acceptance criteria

- the AgentRunner abstraction is clearly defined
- LambdaAgentRunner and AgentCoreRunner are clearly distinguished
- selected_mark_explanation is identified as the first comparison target
- AgentCore is framed as an operational / observability decision, not a response-quality shortcut
- Tableau MCP and LLM follow-up paths are explicitly separated from the runner comparison

## Validation

- docs updated to match the current architecture and roadmap
- follow-up issues are split into PR-sized pieces
