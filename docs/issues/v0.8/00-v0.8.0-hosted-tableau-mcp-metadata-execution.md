# v0.8.0 Hosted Tableau MCP Metadata Execution

## Background

v0.8.0 is the first milestone that should connect a read-only Tableau metadata tool to a gated Hosted / remote Tableau MCP execution path. The v0.7.0 milestone already established the transport, auth, governance, and fake / no-network foundation needed for that step.

## Goal

Implement a gated Hosted / remote Tableau MCP metadata execution path for the first read-only metadata tool, starting with `tableau.metadata.describeDatasource`, while preserving fake / no-network tests, stdio fallback, `selected_mark_explanation` behavior, and the free-form chat legacy path.

## Scope

- planning and Hosted MCP execution boundaries
- Hosted MCP endpoint and site settings verification
- Hosted MCP integration test gating
- HostedTableauMcpTransport or RemoteTableauMcpTransport adapter
- Hosted MCP auth context adapter boundary
- `tableau.metadata.describeDatasource` hosted execution behind feature flag
- Hosted MCP metadata error normalization
- hosted metadata trace coverage
- fake / no-network and stdio fallback tests
- `tableau.metadata.listFields` hosted readiness evaluation
- v0.8.0 wrap-up

## Out of scope

- free-form chat full migration
- `ChatService` replacement
- `runLightweightAgentLoop` replacement
- arbitrary query execution
- underlying data access
- write-capable tools
- raw MCP tool exposure
- broad Tableau MCP tool exposure
- LLM-generated Tableau MCP query execution
- AgentCore implementation
- AgentCoreRunner implementation
- LLM ResponseComposer implementation
- UI trace panel
- trace persistence

## Tasks

- split the milestone into focused child issues
- keep the read-only metadata path first
- define the hosted, auth, and governance boundaries
- preserve the current `selected_mark_explanation` baseline
- document the staged path to v0.9.0

## Acceptance criteria

- v0.8.0 scope is clearly documented
- hosted readiness requirements are documented
- `describeDatasource` has a gated hosted execution path or a documented blocker
- fake / no-network tests continue to pass
- stdio fallback strategy remains available
- hosted integration tests are optional or gated
- hosted metadata errors are normalized into safe summaries
- metadata trace events cover hosted execution
- `selected_mark_explanation` behavior remains unchanged
- free-form chat legacy behavior remains unchanged
- no arbitrary query, underlying data, or write tools are introduced
- the v0.8.0 wrap-up documents whether hosted metadata execution is ready for v0.9.0
