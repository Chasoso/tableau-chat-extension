# v0.7.0 Hosted Tableau MCP Migration Foundation

## Background

v0.7.0 is the first milestone after the v0.6.0 runner spike. The roadmap says this release should prepare `tableau-chat-extension` for a future Hosted / remote Tableau MCP path without attempting a full production migration yet.

## Goal

Prepare `tableau-chat-extension` for migration from current stdio Tableau MCP usage to Hosted / remote Tableau MCP by defining transport boundaries, hosted readiness requirements, read-only metadata tool contracts, fake/no-network execution paths, and migration documentation.

## Scope

- current stdio Tableau MCP usage audit
- Hosted Tableau MCP readiness audit
- Tableau MCP transport abstraction design
- stdio vs hosted transport configuration strategy
- Hosted MCP OAuth / user-context / site-settings boundary
- read-only Tableau metadata tool definitions
- metadata input/output schemas
- metadata preconditions and governance boundaries
- ToolRegistry registration for metadata tools
- fake / no-network metadata handlers
- transport-aware metadata execution boundary
- metadata output normalization and trace events
- migration path documentation from stdio to Hosted Tableau MCP
- v0.7.0 wrap-up

## Out of scope

- full Hosted Tableau MCP production migration
- arbitrary query execution
- Tableau MCP query tools
- underlying data access
- write-capable Tableau MCP tools
- free-form chat full migration
- ChatService replacement
- runLightweightAgentLoop replacement
- LLM ResponseComposer implementation
- Bedrock prompt generation changes
- AgentCore implementation
- AgentCoreRunner implementation
- broad frontend redesign

## Tasks

- split the roadmap into focused child issues
- keep the read-only metadata path first
- define the transport, auth, and governance boundaries
- preserve the current `selected_mark_explanation` baseline
- document the staged migration path to Hosted Tableau MCP

## Acceptance criteria

- v0.7.0 scope is clearly documented
- stdio Tableau MCP responsibilities are audited
- Hosted Tableau MCP readiness requirements are documented
- transport abstraction design is documented
- read-only metadata tool contracts are defined
- metadata tools can be registered with fake/no-network handlers
- metadata execution boundary is transport-aware
- metadata output normalization and trace policy are defined
- migration path from stdio to Hosted Tableau MCP is documented
- existing `selected_mark_explanation` behavior remains unchanged
- free-form chat remains on the legacy path
- no arbitrary query execution is introduced

## Validation

- docs only
- child issue bodies are split into focused follow-up issues
