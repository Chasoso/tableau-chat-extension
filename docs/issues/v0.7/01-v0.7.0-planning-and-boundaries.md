# v0.7.0 planning and boundaries

## Background

The roadmap makes v0.7.0 a foundation milestone, not an implementation sprint. Before any transport work starts, we need a clean plan for what v0.7.0 will and will not do.

## Goal

Finalise the v0.7.0 scope, non-goals, and decision criteria, then turn the roadmap into a v0.7.0-specific plan document.

## Scope

- v0.7.0 plan doc
- scope / non-goals
- child issue mapping
- Hosted Tableau MCP migration assumptions
- safety / governance boundaries
- validation strategy

## Out of scope

- implementation
- transport code
- metadata tool code
- Hosted MCP connection

## Tasks

- restate the v0.7.0 milestone goal in a dedicated plan doc
- map the roadmap items into child issues
- record the assumptions that must stay true for the milestone
- call out the safety and governance boundaries up front

## Acceptance criteria

- the v0.7.0 scope and non-goals are explicit
- the child issue breakdown is clear
- the Hosted MCP assumptions are visible
- the validation path is docs-only

## Validation

- docs only
