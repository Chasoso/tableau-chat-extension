# v0.5.0 Tool Layer Rebuild

## Background

v0.4.0 で structured orchestration の骨組みは整いましたが、ToolRouter と ExecutionEngine はまだ skeleton です。
selected_mark_explanation をはじめとする structured path を実際に道具へ接続するには、ToolDefinition と ToolRegistry の契約が必要です。

## Goal

Tool Layer を MCP / REST API / Notion / context-derived data を含む共通の実行基盤として整理し、selected_mark_explanation に必要な最小の tool path を実行可能にする準備を完了する。

## Scope

- ToolDefinition contract の定義
- ToolRegistry contract の定義
- Tool precondition model の定義
- context-derived pseudo tools の方針整理
- Tableau MCP tools の登録方針整理
- ToolRouter / ExecutionEngine との接続境界整理
- tool execution trace の設計

## Out of scope

- 全 MCP tool の一括移行
- ToolRegistry の大量実装
- LLM による tool selection
- free-form chat の全面移行
- AgentCore
- trace persistence / DB 保存

- ToolDefinition / ToolRegistry の責務が明文化されている
- context-derived pseudo tools の扱いが決まっている
- selected_mark_explanation に必要な最小 tool path が定義されている
- ToolRouter / ExecutionEngine から ToolRegistry に接続できる前提が整理されている
- trace-first 設計が tool execution に拡張できる
