# v0.11.0 Wrap-up

## Summary

v0.11.0 is complete as the milestone that made Hosted Tableau MCP usable in the existing structured metadata discovery flow.

The milestone stayed narrow:

- Hosted execution is opt-in / gated
- `describeDatasource` is the only hosted execution target
- fake / stdio fallback remains available
- default validation remains no-network
- raw MCP output, raw JWTs, raw tokens, and secrets remain hidden

## Milestone result

The milestone outcome is documented in `docs/v0.11-hosted-metadata-discovery-result.md`.

At a high level:

- `metadata_discovery` can now use a gated Hosted `describeDatasource` path for eligible datasource plans
- the implementation keeps the existing flow intact rather than replacing ChatService
- unsupported, ambiguous, and unsafe cases remain outside Hosted execution
- failure modes are normalized safely
- the default path still works without hosted credentials

## What changed

The v0.11.0 issue set delivered:

- #236 planning and scope confirmation
- #237 the code-bearing hosted metadata discovery implementation
- #238 this result and wrap-up documentation

The implementation issue is the actual functional milestone.

## What remained unchanged

v0.11.0 did not change:

- `listFields` execution
- arbitrary query generation or execution
- underlying data access
- field values or row data retrieval
- write-capable tools
- DCR-dependent OAuth 2.1 Remote MCP host flow
- token storage or refresh
- broad free-form chat routing
- broad ChatService replacement
- Exploration Session
- AgentCore
- frontend UI / UX

## Safety posture

The milestone preserved the repository's safety boundary:

- default validation stays no-network
- hosted / external integration tests stay opt-in or gated
- fallback paths remain deterministic
- safe normalization continues to hide raw transport and credential details
- `not_configured`, auth failure, permission denial, timeout, and transport errors are handled safely

## Validation result

The code-bearing implementation in #237 passed the required local checks:

- `npm run quality:precommit`
- `npm run quality:prepush`
- `npm run test:e2e`
- Gitleaks secret scan

This wrap-up issue is docs-only, so it follows the docs review policy rather than adding new runtime validation.

## Remaining limitations

The milestone intentionally leaves these limits in place:

- Hosted execution does not expand beyond `describeDatasource`
- hosted integration tests remain gated
- no broad remote Tableau surface was introduced
- no new runtime routing model was added
- no new secrets handling or token persistence was added

## Next step

If v0.12.0 or a later milestone expands Hosted execution, it should start with a new planning issue and re-state the safety boundary before any implementation work begins.
